## 3. 协程和多线程的介绍及最终的选用

### 3.1  协程

- **概念**：协程是轻量级的线程，允许在单个线程内执行多个任务。通过在任务之间显式地让出控制权（yield 或 await），协程可以在不创建新线程的情况下进行异步操作。

- **特点**：

  单线程：协程通常在单个线程中运行，不需要上下文切换的开销。

  非阻塞：当一个任务在等待某个资源时，协程可以主动切换到其他任务，而不会阻塞整个程序。

  高效：由于不涉及线程创建和销毁的开销，协程比多线程更轻量、资源消耗低。

- **应用**：适用于 I/O 密集型任务，如网络请求、文件读写等。

### 3.2  多线程

- **概念**：多线程是指在一个程序中创建多个线程，每个线程可以并发地执行任务，通常在多核 CPU 上能够实现真正的并行计算。

- **特点**：

  并行执行：多个线程可以在多个 CPU 核心上并行运行，适合 CPU 密集型任务。

  线程切换：操作系统会负责线程的调度和切换，线程间的切换有一定的开销。

  共享资源：多个线程共享内存空间，因此需要小心同步和锁机制，以避免竞态条件和死锁。

- **应用**：适用于计算密集型任务，如并行计算、大规模数据处理等。

### 3.3  对比及最终选用

| 特性               | **协程 (Coroutine)**                                         | **多线程 (Multithreading)**                                  |
| ------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **执行方式**       | 在单线程内通过显式让出控制权（如 `yield` 或 `await`）来切换任务，模拟并发。 | 在多个线程中并行执行任务，通常由操作系统调度。               |
| **创建和销毁开销** | 轻量级，创建和销毁协程的开销非常小。                         | 创建和销毁线程的开销较大，操作系统需要分配和管理线程资源。   |
| **并发/并行**      | 通过异步切换任务，实际只有一个线程执行任务，但可以在等待时切换到其他任务，表现为并发。 | 可以在多核 CPU 上真正实现并行，每个线程独立执行。            |
| **资源占用**       | 协程比线程更轻量，内存占用少，适合大量并发任务。             | 多线程需要更多的内存和 CPU 资源，尤其是线程数量多时。        |
| **适用任务类型**   | 适合 I/O 密集型任务（如网络请求、文件操作），任务间有较长的等待时间。 | 适合 CPU 密集型任务（如并行计算、图像处理），需要多核 CPU 支持。 |
| **难度与复杂度**   | 编程模型简单，代码直观，且可以避免回调地狱（通过 `async/await`） | 编程复杂，尤其需要处理线程同步、死锁和竞态条件等问题。       |

​	因为协程可以高效处理摄像头数据的录制、读取和保存过程，避免I/O 阻塞；协程能在数据保存过程中执行其他任务，提升程序响应速度；协程减少了上下文切换和线程开销，提供了更高效的执行力；并且协程的语法使得代码更加直观、简洁，并且易于维护。综上考虑，最终选用协程的方式。
